# Text field

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for short, editable, single-line text input.
- Use for form fields requiring labels, supporting text, and validation.
- Use outlined or filled variants based on surface emphasis.

## When not to use

- Multi-line freeform text. Use `mx-multiline-text-field`.
- Single-choice selection controls like switches, radios, or segmented buttons.
- Non-editable read-only data display without input intent.

## Accessibility

- **Labeling:** Provide a visible label slot for accessible naming.
- **Support text:** Use support/counter slots for helper and validation guidance.
- **Keyboard:** Input remains focusable and editable with native text-field behavior.
- **Disabled:** `disabled` prevents user editing and interaction.

## Content guidelines

- Keep labels short, specific, and field-oriented.
- Use supporting text for constraints, format hints, or validation context.
- Avoid placeholder-only labeling for essential field meaning.

## Anatomy

- Slots: `label`, `support`, `counter`, `lead`, `trail`, `prefix`, `suffix`.
- CSS parts: `field`, `lead`, `prefix`, `label`, `suffix`, `trail`, `support`, `counter`.

## Theming

- Uses CSS custom properties for field shape, spacing, typography, and states.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
