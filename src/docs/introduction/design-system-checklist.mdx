# Design System Checklist

Use this checklist to keep Material X high quality and release-ready.

1. Purpose and audience are explicit.
2. Foundations (tokens/theming/typography/motion) are documented.
3. Every public component has anatomy, API, states, and examples.
4. Accessibility requirements and caveats are documented.
5. Theming and CSS variable overrides are clear and stable.
6. Framework interoperability is documented (Web Components + React).
7. Docs are source-driven and regenerated in CI.
8. M3 references are included with explicit deltas.
9. Governance for changes/deprecations is documented.
10. Docs publishing pipeline is automated and monitored.
