# Link Button

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for navigation actions that move users to another page or route.
- Use when you need button styling with anchor semantics.
- Use with `target` and external URLs for outbound navigation.

## When not to use

- In-place actions such as submit/save/retry. Use `mx-button`.
- Persistent toggles. Use `mx-switch-button`.
- Icon-only compact actions. Use `mx-icon-button`.

## Accessibility

- **Semantics:** Renders as an anchor element when `href` is provided.
- **Keyboard:** Focusable with `Tab`; activatable via keyboard as a link control.
- **Name:** Visible text in default slot provides the accessible name.
- **Disabled:** `disabled` prevents interaction for disabled link-button states.

## Content guidelines

- Use destination-oriented labels (e.g. “Open docs”, “Go to settings”).
- Avoid verbs that imply immediate in-place execution.
- Pair leading icons with concise destination text when needed.

## Anatomy

- `default` slot: label/content.
- `icon` slot: leading icon content.
- `impl` CSS part: internal anchor implementation.

## Theming

- Uses CSS custom properties for spacing, sizing, and typography overrides.
