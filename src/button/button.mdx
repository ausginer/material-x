# Button

## Summary

{summary}

## When to use

- Use for primary and secondary actions in forms, dialogs, cards, and toolbars.
- Use when users should trigger an immediate action (for example, save, submit, retry).
- Keep one primary action per surface and use other button variants for lower emphasis.

## When not to use

- Navigation to another page or URL. Use `mx-link-button` instead.
- Persistent on/off choices. Use `mx-switch-button` for toggle semantics.
- Dense repeated actions without text labels. Use `mx-icon-button`.

## Accessibility

- **Keyboard:** Focusable with `Tab`; activates with keyboard using native button behavior.
- **Focus indicator:** Uses shared focus styles and delegates focus to the internal button.
- **Name:** The default slot label provides the accessible name.
- **Icon + text:** Use icon as supporting affordance, not a replacement for the text label.
- **Disabled:** `disabled` blocks interaction and form participation.

## Content guidelines

- Use short verb-first labels (e.g. “Save”, “Add item”, “Retry”).
- Avoid ambiguous labels (“OK”) unless context is extremely clear.
- Keep icons meaningful and aligned with the button action.

## Anatomy

- `default` slot: label/content.
- `icon` slot: leading icon content.
- `impl` CSS part: internal native button element.

## Theming

- Uses CSS custom properties for sizing/spacing/typography overrides.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>Toggle Theme</ThemeToggleLink>
