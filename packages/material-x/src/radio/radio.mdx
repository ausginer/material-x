## Summary

## When to use

- Use when users must select exactly one option from a set of two or more mutually exclusive choices.
- Use when all options need to be visible at the same time for comparison.

## When not to use

- Multiple independent selections. Use `mx-checkbox` instead.
- A single on/off choice. Use `mx-checkbox` or `mx-switch-button` instead.
- More than ~6 options. Consider a select/dropdown to save space.

## Grouping

Radio buttons are grouped by the `name` attribute. The browser enforces mutual exclusion and provides arrow-key navigation within the group automatically — no JavaScript wrapper is needed for that behavior.

## Accessibility

- **Keyboard:** `Tab` moves focus to the selected radio in the group (or the first if none selected); arrow keys navigate within the group and select the focused option.
- **Focus indicator:** Drawn around the radio icon (not the full touch target).
- **Name:** Provide an accessible name via a wrapping `<label>` or `aria-label`.
- **Grouping:** Use the same `name` attribute on all radios in a group. The browser provides `role="radiogroup"` semantics automatically.
- **Disabled:** `disabled` blocks interaction and form participation.

## Content guidelines

- List options in a logical order (alphabetical, chronological, or by likelihood).
- Keep labels short and mutually exclusive in meaning.
- Use sentence case for labels.

## Anatomy

- `impl` CSS part: internal native `<input type="radio">` element.

## Theming

- `--md-radio-check-duration` — overrides the selection animation duration.
- `--md-radio-check-easing` — overrides the selection animation easing.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
