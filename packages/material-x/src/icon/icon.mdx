## Summary

## When to use

- Use to support recognition of actions, objects, and states.
- Use alongside text labels for faster visual scanning.
- Use where icon font glyphs provide consistent symbolic language.

## When not to use

- As the only communication for critical or ambiguous actions.
- As decorative noise without functional meaning.
- As a substitute for required explanatory text.

## Accessibility

- **Decorative icons:** Mark with `aria-hidden="true"` when purely decorative.
- **Informative icons:** Provide nearby text or ARIA labeling context.
- **Consistency:** Keep icon semantics stable across screens and components.

## Content guidelines

- Choose universally understood symbols whenever possible.
- Pair icons with text labels for low-confidence or domain-specific glyphs.
- Avoid using too many icon styles in a single flow.
