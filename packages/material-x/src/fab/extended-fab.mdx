# Extended FAB

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use when a promoted action needs both icon and text label.
- Use when the action label should expand/collapse by interaction context.
- Use when directionality or writing mode can affect label presentation.

## When not to use

- Screens that need multiple competing primary actions.
- Tertiary or low-emphasis actions better served by regular buttons.
- Very dense control sets where FAB prominence is excessive.

## Accessibility

- **Name:** The visible extended label should clearly describe the promoted action.
- **Keyboard:** Focusable and activatable with keyboard using native semantics.
- **State:** `extended` and `fabtoggle` represent expanded/collapsed label behavior.
- **Disabled:** `disabled` prevents activation while preserving visual context.

## Content guidelines

- Keep extended label concise and action-oriented.
- Use icons that reinforce the text action rather than replacing it.
- Avoid lengthy labels that break FAB compactness.

## Anatomy

- `default` slot: action label/content.
- `icon` slot: leading icon content.
- `impl` CSS part: internal interactive element.

## Theming

- Uses CSS custom properties for extended/open states and color/size treatment.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
