# FAB

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for the single most important action on a screen.
- Use where a promoted action should remain visually distinctive.
- Use in contexts where icon-led affordance communicates quick action.

## When not to use

- Multiple primary actions competing for attention.
- Low-priority actions that should be standard buttons.
- Dense action sets where toolbar/menus are more appropriate.

## Accessibility

- **Name:** Provide accessible text when icon alone is ambiguous (e.g. `aria-label`).
- **Keyboard:** Focusable and activatable using native button interaction.
- **Focus indicator:** Uses shared focus treatment from core button primitives.
- **Disabled:** `disabled` prevents activation.

## Content guidelines

- Reserve FAB for one clear high-priority action.
- Use widely recognized action icons.
- Keep FAB usage consistent across related surfaces.

## Anatomy

- `default` slot: icon or content.
- `icon` slot: explicit icon placement.
- `impl` CSS part: internal native button element.

## Theming

- Uses CSS custom properties for size, color, tonal, and state overrides.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
