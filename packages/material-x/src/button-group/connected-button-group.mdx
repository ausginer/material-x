# Connected Button Group

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for segmented choices where one selected value should be tracked.
- Use for compact grouped selection controls with connected visual treatment.
- Use when related options should share one selection context (`value`).

## When not to use

- Independent actions without shared selection state. Use `mx-button-group`.
- Single toggle controls. Use `mx-switch-button`.
- Large option sets better represented by lists, tabs, or menus.

## Accessibility

- **Selection model:** Uses shared group `value` to represent the active segment.
- **Keyboard:** Child controls remain focusable and activatable with keyboard input.
- **Labeling:** Every segment should have text or icon labels with accessible names.
- **Disabled:** Group-level `disabled` can block interaction for connected options.

## Content guidelines

- Keep segment labels concise and mutually exclusive.
- Limit option count to maintain quick scanning and selection.
- Keep value semantics stable for predictable selection behavior.

## Anatomy

- `default` slot: connected child controls.
- Host coordinates connected presentation and selected value state.

## Theming

- Uses shared CSS custom properties inherited by child segmented controls.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
