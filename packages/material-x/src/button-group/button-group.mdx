# Button Group

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use to group related actions into one compact horizontal control.
- Use when sibling actions should share size, shape, and color context.
- Use for mutually related commands where order matters visually.

## When not to use

- Single standalone actions. Use `mx-button`.
- Connected/toggleable segmented selections. Use `mx-connected-button-group`.
- Navigation clusters where each action is a route link. Use `mx-link-button`.

## Accessibility

- **Grouping:** Communicates related actions as one grouped control region.
- **Keyboard:** Individual child buttons remain keyboard focusable and operable.
- **Name:** Each child action should have clear visible text or accessible icon labels.
- **Disabled:** Group-level disabled state propagates to child interaction behavior.

## Content guidelines

- Keep action labels short and parallel in phrasing.
- Avoid mixing unrelated actions in the same group.
- Order actions according to usage frequency and user intent flow.

## Anatomy

- `default` slot: grouped button content.
- Group host applies shared visual context to child controls.

## Theming

- Uses CSS custom properties inherited by grouped child buttons.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
