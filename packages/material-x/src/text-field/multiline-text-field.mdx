# Multiline Text Field

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for longer freeform text that spans multiple lines.
- Use in forms requiring notes, descriptions, or messages.
- Use outlined or filled variants based on surface emphasis and hierarchy.

## When not to use

- Single-line input. Use `mx-text-field`.
- Binary or discrete selection controls such as switches and segmented buttons.
- Read-only body text that does not require editing.

## Accessibility

- **Labeling:** Provide a visible label slot to ensure accessible naming.
- **Supporting text:** Use support/counter slots for helper and validation guidance.
- **Keyboard:** Multiline input remains focusable and editable with native textarea behavior.
- **Disabled:** `disabled` prevents editing and interaction.

## Content guidelines

- Prefer concise labels that describe expected long-form input.
- Use supporting text for input constraints and formatting expectations.
- Avoid placeholder-only instructions for required field context.

## Anatomy

- Slots: `label`, `support`, `counter`, `lead`, `trail`, `prefix`, `suffix`.
- CSS parts: `field`, `lead`, `prefix`, `label`, `suffix`, `trail`, `support`, `counter`.

## Theming

- Uses CSS custom properties for field shape, spacing, typography, and state presentation.
