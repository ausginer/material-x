# Checkbox

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use when users need to select one or more independent options from a set.
- Use for binary on/off choices that take effect immediately or on form submit.
- Use `indeterminate` to indicate a partially selected parent state (e.g. "select all" with mixed child states).

## When not to use

- Mutually exclusive choices from a set. Use `mx-radio` instead.
- A single on/off toggle with immediate effect. Use `mx-switch-button` instead.

## Accessibility

- **Keyboard:** Focusable with `Tab`; toggles with `Space`.
- **Focus indicator:** Drawn around the state-layer area (40 × 40 px touch target).
- **Name:** Provide an accessible name via a wrapping `<label>` or `aria-label`.
- **`indeterminate`:** Purely visual — it does not affect the submitted form value. Screen readers announce the element as "mixed". Clear `indeterminate` before toggling `checked` to reflect a definite state.
- **Disabled:** `disabled` blocks interaction and form participation.

## Content guidelines
