# Icon Button

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for compact icon-only actions where a text label is not required.
- Use in dense toolbars, cards, and controls with limited horizontal space.
- Use `width="wide"` or `width="narrow"` when layout requires emphasis tuning.

## When not to use

- Primary actions that need explicit text labels. Use `mx-button`.
- Navigation actions that should use anchor semantics. Use `mx-link-button`.
- Persistent toggles. Use `mx-switch-icon-button`.

## Accessibility

- **Name:** Provide an accessible label with `aria-label` when there is no visible text.
- **Keyboard:** Focusable with `Tab`; activatable with keyboard via native button behavior.
- **Focus indicator:** Uses shared focus visuals through the internal button implementation.
- **Disabled:** `disabled` blocks interaction and form participation.

## Content guidelines

- Use clear, familiar icons with predictable action meaning.
- Do not rely on ambiguous glyphs for destructive or irreversible actions.
- Keep icon choices consistent across related surfaces.

## Anatomy

- `default` slot: icon content.
- `impl` CSS part: internal native button element.

## Theming

- Uses CSS custom properties for sizing, spacing, and icon layout overrides.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
