# Switch Icon Button

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for binary selection with compact icon-only affordances.
- Use in toolbars or segmented controls where text labels are unnecessary.
- Use when selected/unselected state must persist as `checked`.

## When not to use

- One-time icon actions. Use `mx-icon-button`.
- Text-labeled toggles. Use `mx-switch-button`.
- Navigation actions. Use `mx-link-button`.

## Accessibility

- **Name:** Add `aria-label` because visible text labels are absent.
- **Semantics:** Supports toggle state through `checked`, `input`, and `change`.
- **Keyboard:** Focusable and activatable with standard keyboard interaction.
- **Disabled:** `disabled` prevents user interaction and state changes.

## Content guidelines

- Use icons with clear binary state meaning.
- Keep icon metaphors consistent in grouped controls.
- Avoid icons that require supplemental explanatory text.

## Anatomy

- `default` slot: icon content.
- `impl` CSS part: internal interactive element.
