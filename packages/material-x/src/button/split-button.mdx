# Split Button

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use when one primary action and a related secondary menu action must coexist.
- Use where direct click and expanded options are both needed in one control.
- Use for action sets where the default action should stay prominent.

## When not to use

- Single action without secondary choices. Use `mx-button`.
- Pure navigation actions. Use `mx-link-button`.
- Persistent toggles. Use `mx-switch-button` or `mx-switch-icon-button`.

## Accessibility

- **Structure:** Exposes distinct leading/trailing controls through internal parts.
- **Keyboard:** Supports focus and activation using native button keyboard behavior.
- **State:** `open` and `toggle` communicate expanded/collapsed secondary action state.
- **Disabled:** `disabled` disables both split action segments.

## Content guidelines

- Keep primary action text explicit and verb-first.
- Reserve trailing segment for secondary or less frequent choices.
- Use icons as support for action recognition, not replacement for labels.

## Anatomy

- `default` slot: primary label/content.
- `icon` slot: leading icon content.
- CSS parts: `impl`, `group`, `leading`, `trailing`, `leading-impl`, `trailing-impl`.

## Theming

- Uses CSS custom properties for container, split segments, and state styling.
- Works with the global Storybook light/dark theme toggle. <ThemeToggleLink>{THEME_TOGGLE_LABEL}</ThemeToggleLink>
