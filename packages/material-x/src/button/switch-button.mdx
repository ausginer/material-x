# Switch Button

## Summary

<SummaryOrWarning summary={summary} tag={TAG} />

## When to use

- Use for binary on/off states with button visuals.
- Use when pressed state should persist as `checked`.
- Use in segmented actions and settings toggles with text labels.

## When not to use

- One-shot actions. Use `mx-button`.
- Navigation actions. Use `mx-link-button`.
- Icon-only toggles. Use `mx-switch-icon-button`.

## Accessibility

- **Semantics:** Exposes switch-like semantics via `checked`, `input`, and `change`.
- **Keyboard:** Supports focus and activation using native keyboard behavior.
- **Name:** Label text in the default slot provides the accessible name.
- **Disabled:** `disabled` blocks interaction and updates assistive-state signaling.

## Content guidelines

- Use clear binary labels that represent state meaning.
- Keep wording stable across checked and unchecked states.
- Use icons only as supporting cues, not as the sole state indicator.

## Anatomy

- `default` slot: label/content.
- `icon` slot: leading icon content.
- `impl` CSS part: internal interactive element.
